# Protocolo de Diagnostico de Seguridad Raura

## Explicacion ejecutiva del resultado

**Resultado consolidado:** REACTIVO  
**Unidad evaluada:** Raura  
**Base procesada:** 526 registros de respuestas  
**Estructura evaluada:** Liderazgo visible, trabajo en equipo, organizacion y estructura  

Este documento explica el flujo del protocolo de diagnostico, el rol de cada tab del Excel y la razon por la cual el resultado se interpreta como **REACTIVO**. La idea central es que el diagnostico no se sustenta en una sola fuente. El resultado se construye cruzando evidencia documental, observacion en campo, cuestionario, entrevistas y triangulacion.

Un resultado **REACTIVO** significa que la organizacion todavia responde principalmente despues de que aparecen desviaciones, incidentes, incumplimientos o brechas visibles. No significa que no existan actividades de seguridad; significa que las practicas no estan suficientemente integradas, verificadas y sostenidas como sistema preventivo.

## Mensaje para jefatura

El resultado **REACTIVO** se explica porque el protocolo muestra brechas repetidas entre lo que esta documentado, lo que se observa en campo, lo que perciben las personas y lo que se confirma en entrevistas. En otras palabras, existe actividad de seguridad, pero no siempre hay consistencia entre planificacion, ejecucion, seguimiento y cierre.

La conclusion principal es:

> Raura cuenta con elementos de gestion de seguridad, pero el sistema aun depende demasiado de acciones correctivas, esfuerzos individuales y seguimiento posterior. Para avanzar hacia una gestion preventiva, se requiere fortalecer liderazgo visible, disciplina de equipo, claridad de responsabilidades y cierre efectivo de acciones.

## Como se llega al resultado

El protocolo sigue esta cadena:

1. **Se define el marco:** alcance, terminos y criterios de calificacion.
2. **Se levanta evidencia:** documentos, campo, cuestionario y entrevistas.
3. **Se tabulan respuestas:** los datos se ordenan y se transforman en lectura cuantitativa.
4. **Se triangula:** se comparan fuentes para confirmar si una brecha es real, aislada o recurrente.
5. **Se consolida el resultado:** se integran dimensiones y se determina el estado final.

El valor del protocolo esta en la trazabilidad. Cada conclusion debe poder responder tres preguntas:

- Que evidencia lo sostiene.
- En que dimension impacta.
- Que consecuencia tiene para la gestion de seguridad.

# Explicacion por tab

## 1. Inicio

**Funcion del tab:** presentar el contexto general del diagnostico.

Este tab funciona como portada ejecutiva. Debe explicar que se esta evaluando, en que unidad, con que periodo, que dimensiones se analizan y cual es el resultado final.

**Por que es importante para jefatura:** permite entender rapidamente que el diagnostico no es una opinion aislada, sino un proceso estructurado con varias fuentes de informacion.

**Como aporta al resultado REACTIVO:** muestra que el resultado final proviene de un flujo metodologico completo. Si el inicio indica resultado REACTIVO, ese estado debe estar respaldado por resultados, tabulacion y triangulacion.

**Mensaje ejecutivo:**  
El diagnostico se ha estructurado para evaluar si la gestion de seguridad funciona como sistema. La lectura final es REACTIVA porque las evidencias posteriores muestran brechas de consistencia y cierre.

## 2. Abreviaturas

**Funcion del tab:** estandarizar el lenguaje del diagnostico.

Aqui se definen siglas y terminos como LV, trabajo en equipo, organizacion y estructura, evidencia documental, evidencia de campo y triangulacion.

**Por que es importante para jefatura:** evita interpretaciones distintas entre areas. Todos leen los indicadores bajo el mismo criterio.

**Como aporta al resultado REACTIVO:** si no existe lenguaje comun, cada area puede interpretar la seguridad de forma distinta. Esa falta de uniformidad es una senal de gestion reactiva, porque dificulta medir, comparar y corregir.

**Mensaje ejecutivo:**  
Necesitamos un lenguaje unico para que la gestion de seguridad no dependa de interpretaciones personales. El protocolo ordena los conceptos antes de evaluar.

## 3. Criterios de calificacion

**Funcion del tab:** definir las reglas para calificar evidencia, respuestas y brechas.

Este tab responde: cuando consideramos que algo cumple, cumple parcialmente o representa una brecha. Tambien define como se interpreta un resultado bajo, medio o alto.

**Por que es importante para jefatura:** protege la objetividad del diagnostico. Sin criterios claros, el resultado podria parecer subjetivo.

**Como aporta al resultado REACTIVO:** el resultado reactivo aparece cuando las evidencias no alcanzan el nivel esperado de consistencia, seguimiento o madurez. Una actividad puede existir, pero si no tiene cierre, trazabilidad o aplicacion en campo, no debe calificarse como preventiva.

**Mensaje ejecutivo:**  
El resultado no se define por percepcion. Se define comparando evidencias contra criterios. La condicion REACTIVA aparece cuando predominan brechas, controles parciales o acciones correctivas.

## 4. LV Doc y Cam

**Funcion del tab:** evaluar liderazgo visible con evidencia documental y de campo.

Aqui se revisa si los lideres planifican, participan, observan, corrigen y hacen seguimiento en terreno. No basta con tener actas o planes; el protocolo busca confirmar si esas practicas ocurren realmente.

**Por que es importante para jefatura:** el liderazgo visible es uno de los motores principales de cultura de seguridad. Si el liderazgo no se ve en campo, la organizacion tiende a reaccionar tarde.

**Como aporta al resultado REACTIVO:** cuando existe diferencia entre lo documentado y lo observado, el sistema muestra una brecha. Por ejemplo, puede haber registros de inspecciones, pero si no se evidencia retroalimentacion, cierre o presencia constante, el liderazgo no esta operando como barrera preventiva.

**Mensaje ejecutivo:**  
La brecha no es solo si el lider visita campo. La pregunta critica es si esa presencia cambia conductas, corrige desviaciones y asegura cierre. Si no ocurre de forma sostenida, el liderazgo queda en modo reactivo.

## 5. TEquip Doc y Cam

**Funcion del tab:** evaluar trabajo en equipo mediante documentos y observacion en campo.

Este tab revisa coordinacion, reuniones, permisos, comunicacion entre roles, traspaso de informacion y seguimiento de acuerdos.

**Por que es importante para jefatura:** muchas fallas de seguridad no nacen de falta de procedimiento, sino de mala coordinacion entre personas, turnos o areas.

**Como aporta al resultado REACTIVO:** si las reuniones existen pero no generan acuerdos claros, si los permisos no se conectan con controles reales o si el traspaso entre turnos es debil, el equipo trabaja corrigiendo problemas cuando ya aparecen.

**Mensaje ejecutivo:**  
El trabajo en equipo debe verse en rutinas concretas: coordinacion, acuerdos, responsables y cierre. Cuando estas rutinas son incompletas, el control operacional se vuelve reactivo.

## 6. Org. y estruc. Doc y Cam

**Funcion del tab:** evaluar si la estructura organizacional sostiene la gestion de seguridad.

Aqui se revisan roles, responsabilidades, escalamiento, recursos, autoridad para decidir y seguimiento de acciones.

**Por que es importante para jefatura:** una organizacion puede tener buenas intenciones, pero si no tiene responsables claros y rutas de decision efectivas, las brechas se repiten.

**Como aporta al resultado REACTIVO:** el resultado reactivo se refuerza cuando las acciones no tienen propietario claro, cuando el escalamiento es lento o cuando el cierre depende de insistencia personal en vez de un sistema formal.

**Mensaje ejecutivo:**  
La seguridad no puede depender solo del esfuerzo individual. Debe estar sostenida por estructura, roles, autoridad y seguimiento. Si eso no esta maduro, la organizacion reacciona en lugar de prevenir.

## 7. Resultados del Diagnostico

**Funcion del tab:** consolidar la lectura final por dimension y resultado global.

Este tab integra lo encontrado en documentos, campo, cuestionario, entrevistas y triangulacion. Es la hoja que debe explicar el resultado final.

**Por que es importante para jefatura:** traduce datos dispersos en una decision ejecutiva: donde estamos y que debemos priorizar.

**Como aporta al resultado REACTIVO:** el resultado se clasifica como REACTIVO porque las dimensiones evaluadas no muestran suficiente consistencia preventiva. Existen evidencias, pero no todas demuestran cierre, seguimiento, integracion y sostenibilidad.

**Mensaje ejecutivo:**  
El resultado global no debe leerse como una nota aislada. Debe leerse como una conclusion: la gestion existe, pero todavia no opera con madurez preventiva suficiente.

## 8. Tabulacion Cuestionario

**Funcion del tab:** procesar numericamente las respuestas del cuestionario.

Aqui se importan y ordenan las respuestas. Se cuentan alternativas, se calculan totales y se agrupan resultados por pregunta o dimension.

**Por que es importante para jefatura:** permite que la percepcion de las personas se convierta en dato medible. Con 526 registros, la organizacion obtiene una lectura amplia de percepcion y experiencia.

**Como aporta al resultado REACTIVO:** si las respuestas muestran concentracion de percepciones negativas, baja confianza en rutinas o percepcion de debil seguimiento, esas senales refuerzan el resultado reactivo. El cuestionario no define solo el resultado, pero si confirma patrones.

**Mensaje ejecutivo:**  
El cuestionario nos dice como se vive la seguridad en la operacion. Si la percepcion confirma brechas observadas en campo, el resultado reactivo gana solidez.

## 9. Graficos Cuestionar

**Funcion del tab:** convertir la tabulacion en lectura visual.

Los graficos permiten ver tendencias, concentracion de brechas y comparaciones entre dimensiones.

**Por que es importante para jefatura:** facilita tomar decisiones rapidamente. Un grafico debe mostrar donde esta el mayor riesgo de gestion.

**Como aporta al resultado REACTIVO:** cuando los graficos muestran brechas concentradas o bajas valoraciones en dimensiones criticas, se confirma que el problema no es puntual. La tendencia visual ayuda a explicar por que la organizacion debe priorizar acciones correctivas y preventivas.

**Mensaje ejecutivo:**  
Los graficos hacen visible el patron. Si varias preguntas o dimensiones apuntan al mismo problema, ya no hablamos de casos aislados, sino de una condicion de sistema.

## 10. Entrevista Lideres

**Funcion del tab:** recoger evidencia cualitativa desde lideres.

Las entrevistas explican causas que los numeros no muestran: barreras, percepciones, limitaciones de recursos, claridad de roles y nivel de seguimiento.

**Por que es importante para jefatura:** ayuda a entender el "por que" detras de los resultados. No solo dice que hay una brecha; ayuda a explicar su causa.

**Como aporta al resultado REACTIVO:** si los lideres reconocen dificultades de seguimiento, coordinacion, disciplina operacional o cierre de acciones, se confirma que la gestion aun depende de reaccion y correccion posterior.

**Mensaje ejecutivo:**  
Las entrevistas permiten entender si el problema es de conocimiento, liderazgo, estructura o capacidad de ejecucion. Son clave para convertir el diagnostico en plan de accion.

## 11. Liderazgo Visible Triang

**Funcion del tab:** cruzar fuentes para validar liderazgo visible.

Aqui se compara documento, campo, cuestionario y entrevistas. La triangulacion evita concluir solo por una fuente.

**Como aporta al resultado REACTIVO:** si el liderazgo visible aparece documentado pero no se observa consistentemente en campo, o si las entrevistas y cuestionarios muestran baja percepcion de presencia efectiva, la dimension queda en estado reactivo.

**Mensaje ejecutivo:**  
La triangulacion muestra que el liderazgo visible debe pasar de actividad registrada a impacto real en terreno. Esa diferencia explica una parte importante del resultado reactivo.

## 12. Trabajo en equipo Triang

**Mensaje ejecutivo:**  
El trabajo en equipo no se mide por convivencia. Se mide por coordinacion efectiva, acuerdos claros y seguimiento. Si eso falla, el sistema opera reactivamente.

## 13. Org. y estruc. Triang

**Funcion del tab:** cruzar fuentes sobre estructura, responsabilidades y gobierno.

Aqui se verifica si las responsabilidades formales realmente funcionan en la operacion.

**Mensaje ejecutivo:**  
La estructura debe permitir que las acciones se cierren sin depender de empuje personal. Si la estructura no asegura cierre, el sistema no es preventivo.

**Mensaje ejecutivo:**

# Conclusion ejecutiva

El resultado **REACTIVO** se sostiene por la convergencia de varias senales:

- Hay brechas entre documentos y practica real en campo.
- El liderazgo visible requiere mayor consistencia, seguimiento e impacto.
- El trabajo en equipo muestra oportunidades en coordinacion, acuerdos y cierre.
- La organizacion y estructura necesitan mayor claridad de responsables, autoridad y escalamiento.
- El cuestionario y entrevistas ayudan a confirmar que las brechas no son solo documentales, sino percibidas y vividas por las personas.
- La triangulacion permite concluir que varias brechas son recurrentes y sistemicas, no eventos aislados.

La recomendacion para jefatura es enfocar el plan de accion en tres frentes:

1. **Liderazgo visible efectivo:** presencia en campo con retroalimentacion, correccion y cierre.
2. **Rutinas de trabajo en equipo:** reuniones utiles, acuerdos trazables y seguimiento por turno o frente.
3. **Gobierno y estructura:** responsables claros, escalamiento oportuno y cierre sistematico de acciones.

El objetivo no es solo subir un puntaje. El objetivo es mover la gestion desde reaccion y correccion posterior hacia prevencion, aprendizaje y control operacional sostenido.
